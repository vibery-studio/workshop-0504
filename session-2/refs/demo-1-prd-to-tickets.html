PRD → YouTrack tickets — skill + MCP

Paste a PRD, get the tickets. The skill drafts INVEST stories, MCP creates them. About 30 minutes to set up the first time, 30 seconds every time after.

Workshop session 2 · 11/05/2026 · WebMD HS Vietnam
Shape: skill + MCP. The skill knows the ticket format. The MCP server creates them in YouTrack. Same skill works in Claude Code (terminal) and Cowork (browser).

Stack

Surface
Claude Code or Cowork
Skill
~/.claude/skills/prd-to-tickets/
MCP server
YouTrack remote MCP
Auth
YouTrack permanent token

Prerequisites

  • Claude Pro / Max / Team subscription
  • Claude Code installed (or Cowork desktop, macOS/Windows)
  • YouTrack Cloud account + permanent token (Profile → Account Security → New token)

Steps

  1. Create folder ~/.claude/skills/prd-to-tickets/ and add the SKILL.md below.
  2. Add the YouTrack MCP server in Claude Code: 
    claude mcp add youtrack \
  --transport http \
  https://<workspace>.youtrack.cloud/api/mcp \
  --header "Authorization: Bearer <token>"
  3. Open the PRD in your project working directory (markdown file or pasted text).
  4. In Claude Code, type the natural-language trigger from your skill description — e.g. "turn this PRD into tickets." Claude auto-loads the skill.
  5. Review the proposed INVEST-shaped tickets in chat; approve or edit inline.
  6. Confirm — Claude calls YouTrack MCP create_issue for each ticket.
  7. Open YouTrack, filter by today's date, sanity-check.

Minimal SKILL.md

---
name: prd-to-tickets
description: Turn a PRD into INVEST-shaped YouTrack tickets.
  Use when user pastes/links a PRD and asks to create tickets,
  break down a spec, or "make tickets from this PRD".
---

Read the PRD. Extract user stories. For each, write:
- title (≤60 chars)
- description with Acceptance Criteria checklist
- estimate
- labels

Validate INVEST: Independent, Negotiable, Valuable,
Estimable, Small, Testable.

Show the list. Ask user to confirm.
Then call YouTrack MCP create_issue for each ticket.
Gotcha: Skills are not invoked by /slash-commands. They auto-load from the description field. If your description is vague, Claude won't trigger the skill. Make trigger phrases concrete — write the way users actually talk.

If you get it wrong on Monday

  • Skill never fires → description too vague. Add 2–3 specific trigger phrases.
  • YouTrack returns 401 → token expired or wrong workspace URL. Regenerate.
  • Tickets missing labels → SKILL.md doesn't tell Claude which labels exist. Add a list.
  • Same skill in Cowork doesn't see MCP → MCP setup is per-surface; re-add in Cowork settings.
