Design system Q&A — project + MCP

When the answer is in a corpus and the question changes every time. Not a transformation, a lookup. About 15 minutes to set up.

Workshop session 2 · 11/05/2026 · WebMD HS Vietnam
Shape: project + MCP. Two paths — Claude Project for narrative docs (passive corpus, snapshot), or Figma MCP for live design tokens. Pick by question type.

Prerequisites

  • Claude Pro / Team subscription (Projects feature)
  • Design system docs as PDF / DOCX / MD / HTML — tokens, components, voice/tone, accessibility
  • Optional: Figma seat + browser access for OAuth

Path A — Claude Project (narrative docs)

  1. Open claude.ai → Projects → New Project → name it "Design System".
  2. Click "Add knowledge" in the project sidebar. Drag in tokens.md, components.pdf, voice-tone.md, accessibility.md. (Up to 30 MB per file, unlimited count.)
  3. Add a project-level system prompt: 
    Always answer from project knowledge.
Cite the filename.
If not in knowledge, say so.
  4. Open a chat inside the project — every message uses the corpus automatically. No special syntax.
  5. Ask plain questions: "What's our spacing scale?", "Do we have a destructive button variant?"
  6. Citations: Claude shows file pills under answers when content is retrieved.

Path B — Figma MCP (live tokens, optional)

  1. In Claude Code: 
    claude mcp add figma \
  --transport http \
  https://mcp.figma.com/mcp
    Browser will pop OAuth.
  2. Open Figma, select a frame.
  3. Ask Claude: "What tokens does this frame use?"
  4. Claude reads live components, variables, and styles via MCP — no snapshot drift.

Project vs MCP — which when

Use Project for

Voice/tone guides, accessibility principles, onboarding docs, anything narrative. Good when the source updates monthly.

Use Figma MCP for

Tokens, components, color/spacing variables. Good when the source updates daily and you need source-of-truth answers.

Gotcha: Project knowledge is a snapshot — when designers update Figma, the Project files go stale silently. For source-of-truth questions use Figma MCP. For narrative docs use Project. Don't mix them in one project — answers will conflict.

If you get it wrong on Monday

  • Citations not showing → file too small to trigger RAG mode (Claude inlines short knowledge). Add more files.
  • Answers contradict the actual design system → snapshot is stale. Re-upload, or switch to Figma MCP.
  • Designers can't access the Project → Projects are per-user on Pro. Move to Team plan for sharing.
