1:1 prep — pure skill, no MCP

Simplest shape there is. The skill reads me.md, reads a person file, drafts tailored prep notes. No external tools. About 20 minutes to set up.

Workshop session 2 · 11/05/2026 · WebMD HS Vietnam
Shape: pure skill. Just markdown files reading other markdown files. The "wow" is the file structure (me.md + person files), not the skill itself.

Stack

Surface
Claude Code or Cowork
Skill
~/.claude/skills/one-on-one-prep/
Inputs
~/.claude/me.md + 60-people/<name>.md
Optional
Google Calendar connector

Prerequisites

  • Claude Code or Cowork (any subscription tier)
  • Your ~/.claude/me.md from session 1 (or run /me again)
  • One file per direct report at 60-people/<name>.md — role, current goals, last 1:1 notes
  • Optional: Google Calendar connector (Claude Code v2.1.46+, claude.ai → Connectors)

Steps

  1. Create ~/.claude/skills/one-on-one-prep/SKILL.md with the content below.
  2. Make sure ~/.claude/me.md exists. If not, run /me in Claude Code.
  3. For each direct report, create 60-people/<name>.md — one page, role + current goals + last 1:1 notes. (You'll add to these over time.)
  4. Optional: enable Google Calendar connector in claude.ai → Customize → Connectors.
  5. Trigger: "prep my 1:1 with Anh at 3pm" — Claude auto-loads the skill.
  6. The skill reads me.md + the person file via standard file Read tool, no MCP needed.
  7. If Calendar connector enabled, skill pulls today's meeting; otherwise paste the event text.

Minimal SKILL.md

---
name: one-on-one-prep
description: Prep a 1:1 meeting.
  Use when user asks "prep my 1:1", "draft 1:1 agenda",
  or names a direct report + meeting time.
---

1. Read ~/.claude/me.md for the user's working style + rules.
2. Read 60-people/<name>.md if it exists.
3. Get the meeting context: from Google Calendar connector if
   connected, else ask user to paste.
4. Output structure:

   ## Talking points (3)
   - bullet
   - bullet
   - bullet

   ## My ask (1)
   - one ask

   ## Listen for (1)
   - one thing to listen for

Keep it under 200 words. No fluff. Respect the user's
"avoid" rules from me.md (politics, comp, etc.).
Gotcha: A pure-skill demo feels like magic but is just a prompt + file reads. The "wow" is the file structure — me.md + per-person files. If you skip building those first, the skill outputs generic advice. Spend 30 minutes on the people files before you trust the skill.

If you get it wrong on Monday

  • Output is generic → person file is too thin. Add 3 things: their Q-goal, last 1:1's open thread, one rule about how they like feedback.
  • Skill ignores my "avoid politics" rule → me.md rule is in a section the skill didn't read. Move the rule to a top-level "Rules to respect" section.
  • Calendar connector missing → fall back to paste-event-text. Don't block on integrations.
