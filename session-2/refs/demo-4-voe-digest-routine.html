Stack

Surface
Claude Code (local)
Schedule
macOS launchd / cron
Connectors
Sheets · Slack · YouTrack
Output
Slack message + Sheet log

Prerequisites

  • Claude Pro / Max / Team / Enterprise plan
  • Claude Code CLI installed (the runner that fires from cron)
  • Your machine on at 5am — if your laptop sleeps overnight, run on a small server / Raspberry Pi / always-on workstation instead
  • Three MCP connectors authorized in Claude Code:
    • Slack — bot token, channel:write scope on #voe-digest
    • Google Sheets — OAuth, read access to your VoE pulse sheet
    • YouTrack — permanent token, read+write on the relevant project

Steps

  1. Wire the 3 MCP connectors into Claude Code: 
    claude mcp add slack    --transport http https://<slack-mcp>/mcp \
  --header "Authorization: Bearer <slack-bot-token>"
claude mcp add sheets   --transport http https://<sheets-mcp>/mcp \
  --header "Authorization: Bearer <google-token>"
claude mcp add youtrack --transport http https://<workspace>.youtrack.cloud/api/mcp \
  --header "Authorization: Bearer <youtrack-permanent-token>"
  2. Save the skill to ~/.claude/skills/voe-digest/SKILL.md (prompt below).
  3. Create the destination Slack channel #voe-digest. Invite the bot.
  4. Create the Google Sheet "VoE Tracker" with columns: date · theme · signal_count · severity · sample_quote · recommended_step.
  5. Add a cron entry. macOS users: crontab -e and add: 
    0 5 * * 1-5 cd ~ && /usr/local/bin/claude \
  -p "Run voe-digest skill for today" >> ~/voe-digest.log 2>&1
    Or use a launchd plist for a more macOS-native setup. (Set timezone with TZ=Asia/Ho_Chi_Minh.)
  6. Run it once manually to seed the channel and verify auth: claude -p "Run voe-digest skill for today"
  7. Logs land in ~/voe-digest.log — check after every run for the first week.

The 25-line skill prompt

You are a Voice of Employee analyst.

INPUTS (read in order, dedupe across sources):
1. Pulse-survey responses (Sheet "voe-pulse-2026Q2",
   tab "responses", last 24h by submitted_at)
2. Slack #wellness-feedback last 24h (threads + reactions)
3. YouTrack tickets in project ENG with tag customer-signal
   updated last 24h

OUTPUT: one Slack post to #voe-digest

  *Voice of Employee — {date}*

  Top {N} themes from {total_signals} signals.

  *1. {headline}* ({count}, {severity}/5)
     {2-line summary}
     > "{quote_1}"
     > "{quote_2}"
     *Recommended next step:* {one-line action,
     addressed to a function — never a person}

  *2. ...*

RULES:
- Theme = ≥3 signals sharing intent
- Severity 1-5; 5 only if explicit ("I'm leaving")
- Rank by count × severity, cap at 5 themes
- No invented quotes — drop the theme if no real ≤25-word quote
- Match output language to majority of source quotes
- Append a row per theme to "VoE Tracker" Sheet
- If any tool call fails, post error to #ops-alerts
Gotcha: Local schedule means your laptop must stay on at 5am. If it sleeps, the cron skips silently. Three fixes: (1) plug in + disable sleep on AC power overnight; (2) use a launchd StartCalendarInterval with WakeFromSleep; (3) move the cron to a server / Raspberry Pi. Don't trust "I'll just leave the lid open" — macOS still sleeps.
Honest framing: this is automation, not an agent. The prompt is fixed. The schedule is fixed. The MCP servers are fixed. The model only does the judgmental part — clustering noisy signals into themes and writing readable summaries. The mechanical part — reading the Sheet, calling Slack, running on cron — is deterministic code, no AI.

If you get it wrong on Monday

  • Cron didn't fire → laptop slept. Plug in, disable sleep on AC, OR move to launchd with WakeFromSleep.
  • YouTrack returns 401 → permanent token expired or scope too narrow. Regenerate with project-level read+write.
  • Themes are noisy / wrong → severity threshold too low. Bump min count to 4. Or add a "skip if <5 total signals" guard at the top of the prompt.
  • Vietnamese rows ignored → output language defaulting to English. Add explicit rule: "match output language to majority of source quotes."
  • Crisis signals batched into the daily digest → add a "When NOT to use" branch: if resignation cluster ≥5 in 24h, skip digest, post [CRITICAL] to #people-ops-urgent.
  • Wrong tool for the job → if your team's tracker is Linear or Jira, swap the YouTrack connector with the Linear MCP (OAuth-based, drops cleanly into Anthropic Routine today).

When to migrate to Anthropic-hosted Routine

The day JetBrains ships JT-94027:

  1. Re-auth YouTrack via OAuth in claude.ai → Customize → Connectors.
  2. Open claude.ai/code → Routines → New routine. Trigger: scheduled, weekdays preset, 05:00 ICT.
  3. Paste the same skill prompt. Attach 3 connectors. Save.
  4. Disable the local cron entry (don't delete — keep as fallback for two weeks).
  5. Now your laptop closes. Anthropic's cloud runs it. Same digest, same channel.
