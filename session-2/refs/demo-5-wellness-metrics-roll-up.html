Prerequisites

  • Claude Pro / Max / Team / Enterprise plan with Claude Code on the web enabled
  • Daily run quota: Pro 5 / Max 15 / Team-Enterprise 25 (per user, all routines combined)
  • Connectors authorized in claude.ai → Customize → Connectors:
    • Google Sheets — OAuth via Google, read on your operational sheets
    • Notion — OAuth via Notion, write access to your wellness/exec workspace
  • 4–5 operational Sheets that the brief reads from (signups, NPS, completions, churn, MAU — whatever your team tracks)
  • One Notion page where the brief gets posted (e.g. "Wellness — Weekly Brief")

Steps

  1. Authorize both connectors in claude.ai → Customize → Connectors. Each opens a normal OAuth browser flow.
  2. List your operational Sheets in one place — title, URL, what it tracks. Save this list as a small Notion page or a master sheet. The Routine reads this manifest first.
  3. Create the destination Notion page. Title "Wellness — Weekly Brief". Empty for now.
  4. Go to claude.ai/code → Routines → New routine. Name it "Monday Wellness Brief".
  5. Trigger: Scheduled. Pick the "weekly" preset, set time to 07:00 Monday Asia/Ho_Chi_Minh.
  6. Paste the prompt below. Attach both connectors. Save.
  7. First run executes immediately as a smoke test. Open the Notion page — there should be a fresh entry with this Monday's date.
  8. Logs land in the routine's "Runs" tab — full transcript, tool calls, errors.

The 15-line skill prompt

You are a Wellness Program analyst writing a weekly exec brief.

INPUTS — read these Google Sheets (last 7 days):
1. signups — daily new-user signups
2. nps — NPS responses with comments
3. completions — wellness program completion rate
4. churn — cancellations + reasons
5. mau — monthly active users trend

OUTPUT — append a section to the Notion page "Wellness — Weekly Brief":

  ## Week of {monday_date}
  **3 things that moved**
  - {metric, prior → current, % change, one-line why}
  - ...
  **1 risk worth flagging**
  - {a metric trending the wrong way + the leading indicator}
  **1 question for the team**
  - {open question the data raises, addressed to a function — never a person}

RULES:
- No invented numbers. Drop the bullet if the Sheet doesn't have data.
- Match the tone: short sentences, no jargon, no "leverage".
- If any tool call fails, append "[TOOL ERROR: tool_name]" inline.
- Stop at 200 words.
Gotcha: Notion OAuth scopes are workspace-wide by default — your Routine can read every Notion page you have access to. Lock down the Notion integration to the specific workspace (or even page) to avoid accidental data leaks. Same for Sheets — scope to a folder, not your whole Drive.
Honest framing: still not an agent. Fixed prompt, fixed schedule, fixed connectors. The model only does the judgmental part — picking which 3 metrics moved most, writing the "risk worth flagging" line. Reading 5 Sheets and writing to Notion is deterministic plumbing.

If you get it wrong on Monday

  • Notion page has duplicate sections every run → prompt says "append" but routine can re-fire. Add: "If a section for this Monday already exists, replace it."
  • Routine pulls wrong Sheets → manifest list is in your prompt by URL, but you renamed a Sheet. Use Sheet IDs (the long string in the URL) instead of names.
  • Quota exceeded mid-week → Pro is 5 runs/day across ALL routines. Move to Max if you have ~3+ routines.
  • OAuth expired silently → re-auth in Customize → Connectors. Routines don't surface auth-expired errors prominently; check the Runs tab.
  • Brief is too long / generic → add an example brief at the bottom of the prompt. Routines respect format examples better than rules.

Variations to try once it works

  • Switch to Slack output — same data, posted to #exec-wellness instead of Notion.
  • Add HubSpot connector — pull pipeline metrics into the brief.
  • Run daily instead of weekly — change cron to 0 7 * * 1-5, drop the brief into a daily-rolling Notion page.
  • Add a quarterly retrospective Routine — same skill, different schedule, different output page.
