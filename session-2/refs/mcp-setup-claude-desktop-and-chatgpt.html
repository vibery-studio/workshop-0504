Adding MCP — Claude Desktop & ChatGPT

Two consumer apps, two flows, both shipping today (May 2026). Steps below assume you're not an engineer. About 10 minutes per platform.

Workshop session 2 · 11/05/2026 · WebMD HS Vietnam
Quick orientation: Claude Desktop has had MCP since 2024 — Anthropic invented MCP, so support is mature. ChatGPT shipped MCP on March 13, 2026, in beta, behind a "Developer Mode" toggle. Same idea, very different ergonomics.

01Claude Desktop (Anthropic)

Two paths: Custom Connector (remote MCP, recommended for non-engineers) or Local MCP via JSON config (for technical users who want to run servers on their own machine).

Path A — Custom Connector (remote MCP, recommended)

  1. Open Claude Desktop. Click your profile/initials → Settings → Connectors (also reachable via Customize → Connectors).
  2. Scroll to the bottom of the connector list. Click the "+" button → "Add custom connector."
  3. Enter a name (e.g. "Slack") and the remote MCP URL — must be HTTPS, e.g. https://slack-mcp.example.com/mcp.
  4. (Optional) Expand Advanced settings to paste an OAuth Client ID + Secret if the server requires one. Most public connectors don't.
  5. Click Add. Claude opens a browser tab for OAuth — sign in to Slack, grant scopes, return to the app.
  6. In a new chat, click the "+" in the chat box → Connectors → toggle Slack on.
  7. Tools are now callable. Just ask: "Post a message to #voe-digest..."

Path B — Local MCP via JSON config (technical)

Edit the config file directly. Path:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json

Example schema (Slack server via npx):

{
  "mcpServers": {
    "slack": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-slack"],
      "env": { "SLACK_BOT_TOKEN": "xoxb-..." }
    }
  }
}

Save the file → fully quit Claude Desktop (Cmd+Q on macOS, not just close the window) → relaunch. Server now appears in the connector list.

A newer flow: Desktop Extensions (.mcpb bundles) — double-click to install, no JSON. Browse via Settings → Extensions.

Invalid JSON in the config silently disables the server with no error popup; validate with jsonlint before saving. OAuth tokens are stored per-device — signing in on macOS doesn't carry over to your Windows machine.

02ChatGPT (OpenAI)

OpenAI shipped MCP support on March 13, 2026, branded as "MCP connectors" inside Developer Mode (still labeled beta). Available on Plus, Pro, Business, Enterprise, and Edu — but with a sharp split below.

If you're on Plus and the server tries to write to Slack, ChatGPT blocks the call. That's the tier, not your config.

Steps to add a Slack MCP connector

  1. Open ChatGPT (web — chatgpt.com — works first). Click your profile (top-left) → Settings.
  2. Open Apps & Connectors → Advanced Settings.
  3. Toggle Developer Mode on. Read the warning carefully — it's about prompt-injection / data-loss risks. Confirm.
  4. Back to Settings → Connectors → click Create.
  5. Paste the HTTPS MCP URL (must end in /mcp, e.g. https://slack-mcp.example.com/mcp), give it a name + short description, complete OAuth if the server requires it, click Create.
  6. Open a new chat → click the "+" in the composer → More → toggle your Slack connector on.
  7. Prompt normally; ChatGPT will call MCP tools when the request matches a server tool description.
Gotcha: Developer Mode is beta — OpenAI explicitly warns against connecting untrusted servers (write-action mistakes, prompt injection, data exfiltration). Plus/Pro users hitting "tool not available" usually means the server exposes write tools their tier blocks — not a config issue. HTTP (non-HTTPS) URLs are rejected. Mobile and desktop apps lag the web — verify on chatgpt.com before testing in the desktop app. ChatGPT Tasks (scheduled prompts) can trigger MCP tools through enabled connectors, but scheduled runs don't re-prompt for OAuth — re-auth before relying on a scheduled flow.
If you can't use MCP (Free tier or just don't want the complexity):
  • Built-in Connectors — Settings → Connectors → enable Google Drive, Gmail, Calendar, GitHub, SharePoint, Box, Dropbox. One-click OAuth, no URL needed. Best for non-engineers.
  • GPT Actions (Custom GPTs) — Create a Custom GPT → Configure → Actions → Create new action → paste an OpenAPI 3.1 schema describing your API + auth. Heavier setup (you need an OpenAPI spec) but no MCP server required. Works on Plus/Pro/Team.

03Choosing between the two

  • If your team uses Slack + Anthropic: Claude Desktop. Mature, more permissions, no beta warnings.
  • If your team is OpenAI-shop and on Business/Enterprise: ChatGPT MCP works fine — same MCP servers, just a different host.
  • If you're on ChatGPT Plus: read-only is real. For write-heavy workflows (creating tickets, posting messages), use Claude Desktop instead, or upgrade to Business.
  • If you don't want MCP at all: ChatGPT's built-in Connectors cover Drive / Gmail / Calendar / GitHub / SharePoint / Box / Dropbox without any URL or config.
